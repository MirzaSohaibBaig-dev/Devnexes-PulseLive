# Devnexes PulseLive

# Week 1 - Project Planning

---

# 1. Project Overview

Devnexes PulseLive is a real-time polling and audience interaction web application. A host can create live polling sessions and participants can join using a unique session code. Participants cast their votes in real time, and the results are updated instantly without refreshing the page. The application is designed for classrooms, seminars, workshops, meetings, and live events where quick audience feedback is required.

---

# 2. Problem Statement

Traditional methods of collecting audience feedback, such as raising hands or paper-based voting, are slow, difficult to manage, and do not provide instant results. In classrooms, workshops, seminars, and meetings, organizers need a faster and more interactive way to gather responses. A real-time polling application solves this problem by allowing participants to vote instantly while displaying live results to everyone.

---

# 3. Project Objectives

- Create live polling sessions.
- Allow participants to join using a unique session code.
- Support Single Choice, Multiple Choice, and Rating polls.
- Display live voting results.
- Provide a responsive and user-friendly interface.
- Export poll results for future reference.

---

# 4. User Roles

## Host

The Host is responsible for managing the entire polling session.

Responsibilities:

- Create a polling session.
- Generate a unique session code.
- Create polls.
- Start and close polls.
- Reveal live results.
- Export poll results.

---

## Participant

The Participant joins an existing session using the session code.

Responsibilities:

- Join a session.
- Submit votes.
- View live results.
- Wait for the next poll.

---

# 5. Poll Types

### Single Choice Poll

Participants can select only one option from the available choices.

Example:

Which programming language do you prefer?

- Java
- Python
- JavaScript

---

### Multiple Choice Poll

Participants can select more than one option.

Example:

Which technologies have you used?

- React
- Node.js
- Express
- PostgreSQL

---

### Rating Poll

Participants provide ratings using a predefined scale.

Example:

Rate today's workshop.
